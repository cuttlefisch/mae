* Collaborative Setup

This guide walks you through multi-user collaborative editing -- sharing buffers and knowledge bases between MAE instances on different machines.

** 1. Prerequisites

- Two machines (or two MAE instances) on the same network
- Both running MAE v0.11.1 or later
- The =mae-daemon= binary (included in release artifacts)

*Checkpoint:* =:verify= shows version >= 0.11.1.

** Quick Setup

Use =:setup-collab= for interactive configuration, or =:setup-all= to configure everything.

** 2. Start the daemon

On the *host machine*, run:

*** Linux (systemd)
#+begin_src bash
systemctl --user enable --now mae-daemon
#+end_src

*** macOS (Homebrew)
#+begin_src bash
brew services start mae
#+end_src

*** macOS (launchctl)
#+begin_src bash
launchctl load ~/Library/LaunchAgents/com.cuttlefisch.mae-daemon.plist
#+end_src

*** Manual
#+begin_src bash
mae-daemon --bind 0.0.0.0:9473
#+end_src

Note the host's IP address:
- Linux: =hostname -I=
- macOS: =ipconfig getifaddr en0=

*Checkpoint:* Server logs "listening on 0.0.0.0:9473".

** 3. Set a shared secret (PSK)

On *both machines*, add to =~/.config/mae/config.toml=:
#+begin_src toml
[collaboration]
psk = "your-shared-secret-here"
#+end_src

Both machines MUST use the same PSK. For production use, prefer a secrets manager:

*** Linux (pass)
#+begin_src toml
psk_command = "pass show mae/collab-psk"
#+end_src

*** macOS (Keychain)
#+begin_src bash
# Store the PSK:
security add-generic-password -s mae-collab-psk -a mae -w "your-shared-secret"
#+end_src
#+begin_src toml
psk_command = "security find-generic-password -s mae-collab-psk -a mae -w"
#+end_src

The server also needs the PSK in its config (=~/.config/mae/daemon.toml=):
#+begin_src toml
psk = "your-shared-secret-here"
#+end_src

*Checkpoint:* =:describe-configuration= shows =collab_psk: set=.

** 4. Connect from the second machine

#+begin_example
:collab-connect <host-ip>:9473
#+end_example

Or configure auto-connect in =config.toml=:
#+begin_src toml
[collaboration]
server_address = "<host-ip>:9473"
auto_connect = true
#+end_src

*Checkpoint:* =:collab-status= shows =connected=, peer count >= 1.

** 5. Share a buffer

On either machine, open a file and run =:collab-share=. On the other machine, =:collab-list= shows shared docs.

*Checkpoint:* =:collab-status= shows shared doc count >= 1; edits appear on both sides.

** 6. Share a knowledge base

On the host: =:kb-share= -- shares the active KB.
On the peer: =:kb-join= -- joins the shared KB.

*Checkpoint:* =:collab-status= shows KB sync active; =:kb-list= shows shared nodes.

** 7. Verify sync

Edit a shared buffer on one machine -- the change should appear on the other within 1 second. Create a KB node on one machine:
#+begin_example
:kb-create "test-sync" "Sync Test" "note" "Hello from machine A"
#+end_example

*Checkpoint:* =:kb-get "test-sync"= on the other machine returns the node.

** 8. Troubleshooting

Run =:collab-doctor= for comprehensive diagnostics. Common issues:
- *"connection refused"* -- firewall blocking port 9473, or server not running
- *"auth failed"* -- PSK mismatch between machines
- *"timeout"* -- wrong IP address, or machines not on same network

Open firewall if needed:
- macOS: allow in System Settings > Network > Firewall
- Linux: =sudo firewall-cmd --add-port=9473/tcp --permanent=

*Checkpoint:* =:collab-doctor= returns all checks green.
