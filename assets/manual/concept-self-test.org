#+title: AI Self-Test
#+filetags: :ai:testing:tools:

The **self-test** command (`:self-test`) tells the AI agent to exercise its own tool surface and report what works, what's broken, and what's unavailable.

** Usage
- `:self-test` — run all test categories.
- `:self-test introspection` — run only the introspection category.
- `:self-test editing,help` — run multiple specific categories.

** Categories
| Category | What it tests |
|----------|---------------|
| **introspection** | `cursor_info`, `editor_state`, `list_buffers`, `window_layout`, `command_list`, `ai_permissions` |
| **editing** | `create_file`, `buffer_write`, `buffer_read`, `open_file`, `switch_buffer`, `close_buffer` |
| **help** | `kb_search`, `kb_get`, `kb_list`, `kb_graph`, `kb_links_from`, `kb_links_to`, `help_open` |
| **project** | `project_info`, `project_files`, `project_search` (needs git repo) |
| **lsp** | `lsp_diagnostics`, `lsp_document_symbols` (needs LSP server) |
| **dap** | `dap_start`, `dap_set_breakpoint`, `dap_step` (needs lldb-dap or debugpy) |
| **git** | `git_status`, `git_diff`, `git_log`, `git_stash_list` (needs git repo) |
| **modules** | `list_modules`, `describe-module`, module KB nodes |
| **federation** | `kb_instances`, `kb_health`, `concept:kb-federation`, `concept:kb-workflows`, `concept:kb-vs-alternatives` |
| **performance** | `introspect` timing metrics, lock contention, anomaly detection |

** State management
The self-test uses `editor_save_state` before tests and `editor_restore_state` after to leave the editor in a clean state regardless of pass/fail outcomes.

** Reading results
Results appear in the `*AI*` conversation buffer:
- **[PASS]** — tool returned expected data.
- **[FAIL]** — tool returned unexpected data or errored.
- **[SKIP]** — prerequisite not met (e.g. no LSP server).

The self-test also validates the command palette (key commands must exist) and runs a connected help-navigation walkthrough (search → get → graph → open).

** Why this exists
Unit tests validate individual components. The self-test validates the full AI↔editor integration: tool dispatch, permission checks, KB graph integrity, and command registration. It catches wiring bugs that unit tests can't reach.
