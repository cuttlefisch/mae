#+title: Agent Bootstrap
#+filetags: :agents:mcp:ai:

MAE auto-configures AI agents running inside its embedded terminal so they can discover the editor's MCP tools with zero manual setup.

** How it works
1. MAE starts an MCP socket server at `/tmp/mae-{pid}.sock`.
2. The `MAE_MCP_SOCKET` env var is injected into every PTY.
3. On first `:terminal` spawn, MAE writes `.mcp.json` to the project root:
   ```json
   { "mcpServers": { "mae-editor": { "command": "/path/to/mae-mcp-shim" } } }
   ```
4. MAE also writes agent-specific settings to auto-approve tools (e.g. `.claude/settings.local.json` for Claude Code).
5. The agent reads `.mcp.json`, spawns the shim, and gets full tool access.
6. The shim inherits `MAE_MCP_SOCKET` from the shell env and connects.

** Commands
- `:agent-setup <name>` — write `.mcp.json` and approval settings for an agent
- `:agent-list` — show all agents MAE can bootstrap
- `mae --setup-agents [DIR]` — CLI: write configs without starting the editor

** Configuration
In `~/.config/mae/config.toml`:
```toml
[agents]
auto_mcp_json = true       # write .mcp.json on terminal spawn
auto_approve_tools = true  # write agent settings for tool approval
```
Env var overrides: `MAE_AGENTS_AUTO_MCP=0`, `MAE_AGENTS_AUTO_APPROVE=0`

** Adding a new agent
The bootstrap system is agent-agnostic. See the doc comments in `agents.rs` for how to add support for new AI agents. Claude Code is the reference implementation.

** AI permission tiers (internal)
MAE's own tool permissions are separate from agent approval. Use the `ai_permissions` tool or `MAE_AI_PERMISSIONS` env var to control what tier the AI auto-approves up to.
