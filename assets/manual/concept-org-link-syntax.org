#+TITLE: Org Link Syntax — Typed Links, Fragments, and Transclusions

MAE extends standard org-mode link syntax with typed relationships,
fragment addressing, and transclusion directives. These features enable
the knowledge base graph to express rich semantic connections between
nodes.

* Standard Links

The basic link syntax follows org-mode conventions:

- =[[NODE_ID]]= — plain link to a KB node
- =[[NODE_ID][display text]]= — link with custom display text

Examples:

#+begin_example
See [[concept:buffer]] for details.
The [[concept:mode][modal editing system]] is inspired by vi.
#+end_example

* Typed Links

Typed links express the /relationship type/ and metadata of an edge
between two nodes. Relationship metadata lives as *orderless key-value
attributes* in the link target's query string (ADR-030):

#+begin_example
[[NODE_ID?rel=REL_TYPE]]
[[NODE_ID?rel=REL_TYPE][display]]
[[NODE_ID?rel=REL_TYPE&w=0.8&c=0.95][display]]
#+end_example

The grammar for a link *target* is:

#+begin_example
TARGET = NODE_ID [ "#" FRAGMENT ] [ "?" KEY "=" VALUE ( "&" KEY "=" VALUE )* ]
#+end_example

The =NODE_ID= is always clean (e.g. =concept:buffer= — its colons are
part of the id, never a relationship prefix). All edge metadata is in
the =?query=. This is the URI-query model: orderless and extensible.

** Recognized Attributes

| Key                  | Meaning                                  | Default        |
|----------------------+------------------------------------------+----------------|
| =rel=                | relationship type (see table below)      | =references=   |
| =w= (or =weight=)    | edge weight, clamped to 0–1              | =1.0=          |
| =c= (=conf=/=confidence=) | edge confidence, clamped to 0–1     | =1.0=          |

Attributes are *orderless* — =?rel=teaches&w=0.8= and =?w=0.8&rel=teaches=
are equivalent. A malformed numeric value falls back to its default
(it never drops the link).

** Custom / Future Attributes

Any key the parser does not recognize is *preserved verbatim* (in source
order) and ignored by the graph today — so new edge attributes need no
grammar change:

#+begin_example
[[concept:buffer?rel=cites&since=2026-06&by=ai&strength=hard][buffers]]
#+end_example

Here =rel= is recognized; =since=, =by=, and =strength= round-trip as
custom attributes that future tooling can read. The AI peer authoring
enrichment links should follow the same grammar.

** Usage in Org Files

Typed links appear inline in the body text, just like standard links:

#+begin_example
This lesson teaches the fundamentals of buffer navigation.
See [[concept:buffer?rel=teaches]] and [[concept:window?rel=teaches]].

Prerequisites: [[lesson:modes?rel=requires]].
#+end_example

When the org parser encounters a typed link, it:
1. Extracts the rel_type (e.g., =teaches=)
2. Extracts the target node ID (e.g., =concept:buffer=)
3. Stores both in the =typed_links= field of the parse result
4. The link is stored in CozoDB with the correct =rel_type=

* Fragment Links

Fragment links address a specific section or block within a node:

- =[[NODE_ID#heading-slug]]= — named heading fragment
- =[[NODE_ID#3]]= — numeric block index (0-based)
- =[[NODE_ID#fragment?rel=REL_TYPE]]= — typed link with fragment
  (the =#fragment= comes /before/ the =?query=)

Examples:

#+begin_example
See [[concept:buffer#rope-internals]] for the data structure.
Block 3 covers cursors: [[concept:mode#3]].
This section teaches: [[concept:scheme-api#hooks?rel=teaches]].
#+end_example

Fragment navigation:
- Named fragments match heading text (slugified, case-insensitive)
- Numeric fragments jump to the Nth blank-line-separated block
- The fragment is stripped from the target ID for node lookup

* Transclusion Directives

Transclusion includes content from another node inline:

=#+TRANSCLUDE: NODE_ID ROLE=

Roles:
- =content= — include the full body text
- =reference= — show only a link to the node
- =transclusion= — inline the body within the current node

Example:

#+begin_example
#+TRANSCLUDE: concept:buffer-view content
#+TRANSCLUDE: concept:rope-internals reference
#+end_example

Transcluded nodes become members of the containing meta-node.
The =compose_meta_body= function assembles the final text.

* Property Drawer

Every org file in =assets/manual/= uses a property drawer:

The property drawer uses ~:ID:~ for the node identifier, ~:KIND:~ for
the node type, and ~:ALIASES:~ for search aliases:

~:ID: concept:buffer~, ~:KIND: concept~, ~:ALIASES: rope, text buffer~

The =:KIND:= property maps to [[concept:knowledge-base][NodeKind]]:
=concept=, =lesson=, =tutorial=, =key=, =note=, =command=, =category=,
=meta=, =block=, =scheme_api=, =task=, =view=.
